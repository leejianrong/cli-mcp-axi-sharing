A mini CI/CD pipeline service

This is the dummy app we built: a tiny, fully offline stand-in for something like GitLab CI or GitHub Actions. It stores pipeline runs, each with a branch, a commit, and a set of jobs (lint · unit · e2e) that pass, fail, or get skipped. The same data is exposed three ways: a verbose CLI, an MCP server, and a principled AXI command. The benchmark below runs one task through each front door and measures what it costs. This dashboard is just the data all three are looking at.
