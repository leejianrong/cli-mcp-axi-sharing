# Planning docs (archived)

These are the docs that shaped the talk, kept for background. They are not the live
artifacts, so treat them as history rather than source of truth:

- `discovery_notes.md` — the aligned plan, locked defaults, and honesty framing.
- `presentation_outline.md` — the slide-by-slide outline.
- `live_demo_script.md` — the on-stage runbook and fallback plan.

What's current lives one level up: the read-off script is
[`../presentation_script.md`](../presentation_script.md), and the deck itself is in
[`../../slides/`](../../slides/) (deployed to GitHub Pages). Where these archived docs
disagree with the slides or the code, the slides and code win.
